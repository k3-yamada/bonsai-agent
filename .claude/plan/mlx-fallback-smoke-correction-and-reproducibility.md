### TODO #3: Smoke Correction Coefficient

PASS は全項目達成:

| Signal | Acceptance Criteria |
|--------|---------------------|
| Unit tests (smoke off) | `apply_smoke_correction_to_delta(0.10) == 0.10` (delta 不変) |
| Unit tests (smoke on, default) | `apply_smoke_correction_to_delta(0.10) ≈ 0.042` (×0.42) |
| Unit tests (env override valid) | `BONSAI_LAB_SMOKE_CORRECTION=0.5` で `0.10 → 0.05` |
| Unit tests (env override invalid) | `"abc"`, `"0"`, `"-1"`, `"1.5"`, `"NaN"` で 0.42 fallback、panic なし |
| Unit tests (sign-aware) | negative delta は scaling せず: `apply_smoke_correction_to_delta(-0.10) == -0.10` (smoke on でも) |

#### Step 4.2: Green — Implementation (20min)

`src/agent/experiment.rs` の `estimate_mutation_effect_with_baseline` (line 506 周辺) 直前に追加:

```rust
/// Lab smoke モード判定（既存 BONSAI_LAB_SMOKE と同セマンティクス）
fn lab_smoke_enabled() -> bool {
    std::env::var("BONSAI_LAB_SMOKE")
        .map(|v| !v.is_empty() && v != "0")
        .unwrap_or(false)
}

/// smoke 補正係数 — 項目 184 の smoke→core 42% retention 由来
const DEFAULT_SMOKE_CORRECTION: f64 = 0.42;

fn smoke_correction_coefficient() -> f64 {
    std::env::var("BONSAI_LAB_SMOKE_CORRECTION")
        .ok()
        .and_then(|v| v.trim().parse::<f64>().ok())
        .filter(|v| v.is_finite() && *v > 0.0 && *v <= 1.0)
        .unwrap_or(DEFAULT_SMOKE_CORRECTION)
}

/// smoke モード時に推定 delta に補正係数を適用（sign-aware: positive のみ scaling）
///
/// **Why sign-aware**: smoke が positive delta を inflate するなら scaling は妥当だが、
/// negative delta も同じ係数で scaling すると false-accept リスク (-0.10 → -0.042 で
/// -0.01 threshold を通過) が発生する。inflated win のみ補正、negative signal は保持。
fn apply_smoke_correction_to_delta(delta: f64) -> f64 {
    if lab_smoke_enabled() && delta > 0.0 {
        delta * smoke_correction_coefficient()
    } else {
        delta
    }
}
```

- handoff に最終 RSS を記録、長期トレンド監視
